# Chess

A polished, native cross-platform chess game.

- **Stack:** Tauri 2 (Rust) + Svelte 5 + TypeScript
- **AI:** Hybrid — custom Rust minimax engine for difficulty 1–3, bundled Stockfish 17 (UCI sidecar) for 4–10
- **Modes:** Human vs Human (local), Human vs AI
- **Platforms:** macOS, Windows, Linux — all installers built natively from macOS

## Project layout

| Path | Owner | Contents |
| --- | --- | --- |
| `src/` | frontend subagent | Svelte 5 UI, SVG piece sets, panels, modals, audio, stores, mock API |
| `src-tauri/src/api.rs` | shared contract | Rust types — single source of truth for the IPC surface |
| `src-tauri/src/commands.rs` | shared contract | Tauri command signatures (frontend → backend) |
| `src-tauri/src/{engine,ai,pgn,clock,session}` | backend subagent | Chess rules, AI, PGN, clock, session manager |
| `src-tauri/binaries/` | – | Stockfish sidecar binaries (downloaded by `scripts/fetch-stockfish.sh`) |
| `scripts/` | – | Cross-platform packaging from macOS |

The frontend builds against the auto-generated TypeScript bindings at
`src/lib/api/bindings.ts` (regenerated by `tauri-specta` on every backend
build). When `bindings.ts` is empty (fresh checkout, no backend build yet),
the hand-mirrored `src/lib/api/contract.ts` provides identical types. A
browser-only mock implementation (`src/lib/api/mock.ts`) lets the frontend
run via plain `pnpm dev` without the Rust backend.

## Develop

```bash
pnpm install                   # install JS deps
pnpm dev                       # frontend only, mock backend (in browser)
pnpm tauri:dev                 # full app, real Rust backend
pnpm check                     # type-check Svelte + TS
```

## Generate icons (one-time, from the official source PNG)

```bash
pnpm icons                     # writes platform icons into src-tauri/icons/
```

## Build installers from macOS

One-time setup for cross-compilation:

```bash
# Windows
brew install nsis llvm
echo 'export PATH="/opt/homebrew/opt/llvm/bin:$PATH"' >> ~/.zshrc
rustup target add x86_64-pc-windows-msvc aarch64-apple-darwin x86_64-apple-darwin
cargo install --locked cargo-xwin

# Linux
brew install --cask docker     # then start Docker.app

# Stockfish (one-time download of all four sidecar binaries)
bash scripts/fetch-stockfish.sh
```

Build:

```bash
pnpm dist:mac                  # -> dist/installers/macos/Chess_*.dmg
pnpm dist:win                  # -> dist/installers/windows/Chess_*.exe (NSIS)
pnpm dist:linux                # -> dist/installers/linux/Chess_*.AppImage + .deb
pnpm dist:all                  # all three
```

### Caveats

- macOS `.app`/`.dmg` is unsigned (no Apple Developer ID configured). Users will need to right-click → Open. To enable signing, fill in `scripts/sign-mac.sh` and re-run.
- Windows `.exe` is unsigned (no code-signing cert configured). SmartScreen will warn on first launch.
- Linux MSI is intentionally not built from macOS — Tauri's docs note WiX/MSI requires a Windows host. NSIS `.exe` is the standard cross-host path and works.

## Architecture

Two subagents own the two halves of the app and communicate only via the
typed Tauri command/event surface defined in `src-tauri/src/api.rs`.

```text
                    ┌─────────────────────────────┐
                    │   Svelte 5 Frontend (TS)    │
                    │                             │
                    │  Board • Pieces • Panels    │
                    │  Modals • Audio • Stores    │
                    └──────────────┬──────────────┘
                                   │ invoke / events
                                   │ (typed via tauri-specta)
                    ┌──────────────┴──────────────┐
                    │     Rust Backend (Tauri)    │
                    │                             │
                    │  engine • ai • pgn • clock  │
                    │  session • commands         │
                    │             │               │
                    │             └─► Stockfish   │
                    │                 (sidecar)   │
                    └─────────────────────────────┘
```
