# Chess

A polished, native cross-platform chess game.

- **Stack:** Tauri 2 (Rust) + Svelte 5 + TypeScript
- **AI:** Hybrid — custom Rust minimax engine for difficulty 1–3, bundled
  Stockfish 17 (UCI sidecar) for 4–10. Falls back to the custom engine
  if the Stockfish binary is missing or fails to spawn.
- **Modes:** Human vs Human (local), Human vs AI
- **Platforms:** macOS, Windows, Linux

## Project layout

| Path | Contents |
| --- | --- |
| `src/` | Svelte 5 UI, SVG piece sets, panels, modals, audio, stores, mock API |
| `src-tauri/src/api.rs` | Shared Rust↔TS contract (single source of truth) |
| `src-tauri/src/commands.rs` | Tauri command surface (frontend → backend) |
| `src-tauri/src/{engine,ai,pgn,clock,session}` | Chess rules, AI, PGN, clock, session manager |
| `src-tauri/src/lib.rs` | App entry + `tauri-specta` bindings export |
| `src-tauri/examples/export_bindings.rs` | Headless TS-bindings regenerator |
| `src-tauri/binaries/` | Stockfish sidecar binaries (downloaded by `scripts/fetch-stockfish.sh`, not committed) |
| `src-tauri/tests/integration.rs` | End-to-end backend smoke test (no Tauri runtime needed) |
| `scripts/` | Cross-platform packaging helpers |

The frontend builds against the auto-generated TypeScript bindings at
`src/lib/api/bindings.ts` (regenerated by `tauri-specta` on every debug
build, or manually via `cargo run --example export_bindings`). When
`bindings.ts` is empty (fresh checkout, no backend build yet), the
hand-mirrored `src/lib/api/contract.ts` provides identical types. A
browser-only mock implementation (`src/lib/api/mock.ts`) lets the
frontend run via plain `pnpm dev` without the Rust backend.

## Develop

```bash
pnpm install                   # install JS deps
pnpm dev                       # frontend only, mock backend (in browser)
pnpm tauri:dev                 # full app, real Rust backend
pnpm check                     # type-check Svelte + TS
cd src-tauri && cargo test     # run engine + integration tests
```

To regenerate the TypeScript bindings without launching the GUI (useful
in CI / headless environments):

```bash
cd src-tauri
cargo run --example export_bindings
# or to a custom path:
cargo run --example export_bindings -- ../src/lib/api/bindings.ts
```

## Generate icons (one-time, from the official source PNG)

```bash
pnpm icons                     # writes platform icons into src-tauri/icons/
```

## Build installers

The Stockfish 17 sidecar is the same on every host; download the four
target binaries first. They are gitignored and **must be re-downloaded
on a fresh checkout**:

```bash
bash scripts/fetch-stockfish.sh    # writes src-tauri/binaries/stockfish-*
```

### Build matrix (verified)

| Host \\ Target | Linux x86_64 (`.deb`, `.AppImage`) | Windows x86_64 (`.exe` NSIS) | macOS (`.dmg`) |
| --- | --- | --- | --- |
| **Linux x86_64** | ✅ native | ✅ via `cargo-xwin` | ❌ requires macOS |
| **macOS arm64 / x86_64** | ✅ via Docker | ✅ via `cargo-xwin` + brew NSIS | ✅ native |

#### Linux (build on Linux host)

One-time setup:

```bash
sudo apt-get update && sudo apt-get install -y \
  build-essential curl wget file libssl-dev pkg-config libgtk-3-dev \
  libwebkit2gtk-4.1-dev libayatana-appindicator3-dev librsvg2-dev
rustup target add x86_64-unknown-linux-gnu
cargo install --locked tauri-cli
```

Build (or run `pnpm dist:linux` which calls the same commands):

```bash
pnpm tauri build --bundles deb,appimage --target x86_64-unknown-linux-gnu
# Outputs:
#   src-tauri/target/x86_64-unknown-linux-gnu/release/bundle/deb/Chess_*.deb
#   src-tauri/target/x86_64-unknown-linux-gnu/release/bundle/appimage/Chess_*.AppImage
```

#### Windows (cross-compile from Linux or macOS)

One-time setup:

```bash
# Linux:  sudo apt-get install -y nsis llvm clang lld
# macOS:  brew install nsis llvm; export PATH="/opt/homebrew/opt/llvm/bin:$PATH"
rustup target add x86_64-pc-windows-msvc
cargo install --locked cargo-xwin tauri-cli
```

Build (or run `pnpm dist:win`):

```bash
pnpm tauri build --runner cargo-xwin --target x86_64-pc-windows-msvc
# Output:
#   src-tauri/target/x86_64-pc-windows-msvc/release/bundle/nsis/Chess_*-setup.exe
```

> The first run downloads the Windows SDK headers (~500 MB, cached
> under `~/.cache/cargo-xwin/`). The Tauri CLI's `--bundles nsis` flag
> is rejected on non-Windows hosts (the host-side filter only
> recognises Linux bundle types); we omit it and rely on
> `bundle.targets="all"` in `tauri.conf.json`, which selects NSIS for
> Windows targets. MSI is intentionally not produced cross-host (Tauri
> needs a Windows host for WiX).

#### macOS (build on macOS host only)

```bash
pnpm dist:mac                  # → dist/installers/macos/Chess_*.dmg
```

## How to install on each platform

- **Linux (Debian/Ubuntu):** `sudo dpkg -i Chess_0.1.0_amd64.deb`
  (then resolve any GTK/WebKit deps with `sudo apt-get -f install`).
- **Linux (any distro):** `chmod +x Chess_0.1.0_amd64.AppImage && ./Chess_0.1.0_amd64.AppImage`.
- **Windows:** double-click `Chess_0.1.0_x64-setup.exe` and follow the
  installer. SmartScreen will warn on first launch (the binary is
  unsigned); choose "More info" → "Run anyway".
- **macOS:** open the `.dmg` and drag Chess into Applications. The app
  is unsigned, so right-click → Open the first time to bypass
  Gatekeeper.

## Testing

```bash
cd src-tauri
cargo test --package chess              # engine perft + invariants
cargo test --test integration -- --nocapture
cargo clippy --all-targets -- -D warnings
```

The integration test exercises `SessionManager` + commands + AI + PGN
end-to-end without needing a live Tauri runtime, including a graceful
Stockfish-fallback path.

## Known caveats

- Installers are **unsigned** on every platform. macOS Gatekeeper and
  Windows SmartScreen will warn on first launch. Sign manually with a
  Developer ID / EV cert if you ship to end users.
- The bundled Stockfish 17 binary is ~80 MB per platform; that's why
  the .deb is ~67 MB and the .AppImage is ~140 MB.
- See `KNOWN_ISSUES.md` for behavioural caveats discovered during
  integration testing.
